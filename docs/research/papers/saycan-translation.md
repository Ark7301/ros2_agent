## 译文

### 做我能做的，而非我所说的：在机器人可供性中落地语言模型

#### 摘要

大型语言模型（LLM）能够编码关于世界的丰富语义知识。这些知识对于希望执行用自然语言表达的高层次、时间扩展指令的机器人来说极其有用。然而，语言模型的一个显著弱点是它们缺乏真实世界的经验，这使得难以将其用于特定具身体的决策。例如，询问语言模型如何清理溢出物可能会得到合理的叙述，但这可能不适用于需要在特定环境中执行此任务的特定智能体（如机器人）。

我们提出通过预训练技能提供真实世界的落地能力，这些技能用于约束模型提出既可行又符合上下文的自然语言动作。机器人可以充当语言模型的"手和眼睛"，而语言模型则提供关于任务的高层次语义知识。我们展示了如何将低层次技能与大型语言模型结合，使语言模型提供执行复杂和时间扩展指令的程序性高层次知识，而与这些技能相关的价值函数提供将这些知识连接到特定物理环境所需的落地能力。

我们在多个真实世界机器人任务上评估了该方法，展示了真实世界落地的必要性，以及该方法能够在移动操作机器人上完成长时域、抽象的自然语言指令。

#### 1 引言

大型语言模型（LLM）的最新进展已经产生了能够基于提示生成复杂文本、回答问题或就广泛主题进行对话的系统。这些模型从网络挖掘的文本语料库中吸收了大量知识，我们可能会想知道这些模型中编码的日常任务知识是否可以被机器人用于在真实世界中执行复杂任务。但具身智能体如何提取和利用 LLM 的知识来完成物理落地的任务？

这个问题构成了一个重大挑战。LLM 并未在物理世界中落地，它们不会观察其生成对任何物理过程的后果。这可能导致 LLM 不仅犯下对人类来说不合理或可笑的错误，还会以对特定物理情况无意义或不安全的方式解释指令。图 1 展示了一个例子——一个能够执行"拿起海绵"或"去桌子"等技能的厨房机器人可能被要求帮助清理溢出物（"我洒了饮料，你能帮忙吗？"）。语言模型可能会回应一个合理但对机器人不可行或无用的叙述。"你可以尝试使用吸尘器"在场景中没有吸尘器或机器人无法使用吸尘器时是不可能的。

受此例子启发，我们研究如何提取 LLM 中的知识，使具身智能体（如机器人）能够遵循高层次文本指令。机器人配备了一套用于"原子"行为的学习技能，这些技能能够进行低层次视觉运动控制。我们利用这样一个事实：除了简单地要求 LLM 解释指令外，我们还可以使用它来评分单个技能对完成高层次指令的贡献可能性。然后，如果每个技能都有一个可供性函数来量化它从当前状态成功的可能性（例如学习的价值函数），其值可以用来加权技能的可能性。通过这种方式，LLM 描述每个技能对完成指令的贡献概率，可供性函数描述每个技能成功的概率——将两者结合提供每个技能成功执行指令的概率。

#### 2 预备知识

**大型语言模型**：语言模型寻求对文本 W = {w₀,w₁,w₂,...,wₙ}（字符串 w 的序列）的概率 p(W) 进行建模。这通常通过链式法则分解概率来完成。近期基于神经网络的注意力架构的突破使得所谓的大型语言模型（LLM）能够高效扩展。这些模型包括 Transformers、BERT、T5、GPT-3、Gopher、LAMDA、FLAN 和 PaLM，每个都显示出越来越大的容量（数十亿参数和数 TB 文本）以及随后跨任务泛化的能力。

在本工作中，我们利用 LLM 中包含的丰富语义知识来确定解决高层次指令的有用任务。

**价值函数与强化学习**：我们的目标是能够准确预测技能（由语言命令给出）在当前状态下是否可行。我们使用基于时序差分（TD）的强化学习来实现这一目标。特别地，我们定义马尔可夫决策过程（MDP）M = (S,A,P,R,γ)，其中 S 和 A 是状态和动作空间，P 是状态转移概率函数，R 是奖励函数，γ 是折扣因子。

TD 方法的目标是学习状态或状态-动作价值函数（Q 函数）Qᵖ(s,a)，它表示从状态 s 和动作 a 开始，然后遵循策略 π 产生的动作时的折扣奖励总和。在本工作中，我们利用基于 TD 的方法来学习额外以语言命令为条件的价值函数，并利用这些函数来确定给定命令在给定状态下是否可行。

#### 3 SayCan：做我能做的，而非我所说的

**问题陈述**：我们的系统接收用户提供的自然语言指令 i，描述机器人应执行的任务。指令可以是长的、抽象的或模糊的。我们还假设给定一组技能 Π，其中每个技能 π ∈ Π 执行一个短任务（如拾取特定对象），并附带简短的语言描述 ℓπ（例如"找到海绵"）和可供性函数 p(cπ|s,ℓπ)，它指示从状态 s 成功完成描述为 ℓπ 的技能的概率。

**将大型语言模型连接到机器人**：虽然大型语言模型可以利用从大量文本中学到的丰富知识，但它们不一定会将高层次命令分解为适合机器人执行的低层次指令。如果询问语言模型"机器人如何给我带来苹果"，它可能会回答"机器人可以去附近的商店为你购买苹果"。尽管这个回答对提示来说是合理的完成，但对于可能具有狭窄且固定能力集的具身智能体来说，它不一定是可操作的。

因此，为了使语言模型适应我们的问题陈述，我们必须以某种方式告知它们，我们特别希望将高层次指令分解为可用低层次技能的序列。一种方法是仔细的提示工程，这是一种诱导语言模型产生特定响应结构的技术。提示工程在上下文文本（"提示"）中为模型提供示例，指定任务和模型将模仿的响应结构。

**SayCan**：SayCan 的关键思想是通过价值函数——捕获特定技能在当前状态下成功可能性的对数似然的可供性函数——来落地大型语言模型。给定技能 π ∈ Π、其语言描述 ℓπ 及其对应的价值函数（提供 p(cπ|s,ℓπ)，即在状态 s 中由 ℓπ 描述的技能完成的概率），我们形成可供性空间 {p(cπ|s,ℓπ)}π∈Π。

对于每个技能，可供性函数和 LLM 概率然后相乘，最终选择最可能的技能，即 π = argmaxπ∈Π p(cπ|s,ℓπ)p(ℓπ|i)。一旦选择了技能，相应的策略就由智能体执行，LLM 查询被修改以包含 ℓπ，并再次运行该过程，直到选择终止标记（例如"完成"）。

这两个镜像过程共同导致 SayCan 的概率解释，其中 LLM 提供技能对高层次指令有用的概率，可供性提供成功执行每个技能的概率。将这两个概率结合在一起，提供了该技能推进用户命令的高层次指令执行的概率。

#### 4 在机器人系统中实现 SayCan

**语言条件机器人控制策略**：为了实例化 SayCan，我们必须为其提供一组技能，每个技能都有策略、价值函数和简短的语言描述（例如"拾起罐子"）。在我们的实现中，我们使用基于图像的行为克隆（遵循 BC-Z 方法）或强化学习（遵循 MTOpt）来训练各个技能。无论如何获得技能的策略，我们都利用通过 TD 备份训练的价值函数作为该技能的可供性模型。

为了分摊训练许多技能的成本，我们分别利用多任务 BC 和多任务 RL，其中我们不是为每个技能训练单独的策略和价值函数，而是训练以语言描述为条件的多任务策略和模型。

**训练低层次技能**：我们利用 BC 和 RL 策略训练程序分别获得语言条件策略和价值函数。如前所述，对于技能规范，我们使用一组简短的自然语言描述，这些描述表示为语言模型嵌入。我们使用稀疏奖励函数，如果语言命令成功执行，则在回合结束时奖励值为 1.0，否则为 0.0。

**机器人系统和技能**：对于控制策略，我们使用移动操作机器人研究了一组多样化的操作和导航技能。受人们可能在厨房环境中向机器人提出的常见技能启发，我们提出了 551 个技能，涵盖 7 个技能家族和 17 个对象，包括拾取、放置和重新排列对象、打开和关闭抽屉、导航到各个位置以及将对象放置在特定配置中。

#### 5 实验评估

**实验设置**：我们在两个办公室厨房环境中使用移动操作机器人和一组对象操作和导航技能评估 SayCan。我们使用 15 个办公室厨房中常见的对象和 5 个具有语义意义的已知位置（两个柜台、一张桌子、一个垃圾桶和用户位置）。我们在两个环境中测试我们的方法：一个真实的办公室厨房和一个镜像厨房的模拟环境，这也是训练机器人技能的环境。使用的 LLM 是 540B PaLM，除非另有说明用于 LLM 消融。我们将使用 PaLM 的 SayCan 称为 PaLM-SayCan。

**指令**：为了评估 PaLM-SayCan，我们测试了来自 7 个指令家族的 101 条指令。这些指令跨越多个变化轴：时间范围（从单个原语到连续 10+ 个）、语言复杂性（从结构化语言到完全众包请求）和具身性（机器人和环境状态的变化）。

**指标**：为了理解所提方法的性能，我们测量两个主要指标。第一个是规划成功率，它衡量模型选择的技能对于指令是否正确，无论它们是否实际成功执行。第二个指标是执行成功率，它衡量完整的 PaLM-SayCan 系统是否实际成功执行了所需的指令。


#### 5.1 结果

表 2 显示了 PaLM-SayCan 在 101 个任务上的性能。在模拟厨房中，PaLM-SayCan 实现了 84% 的规划成功率和 74% 的执行率。我们还在真实厨房环境中调查 PaLM-SayCan，以验证策略和价值函数在此设置中的性能。我们发现规划性能降低了 3%，执行降低了 14%，表明 PaLM-SayCan 和底层策略能够合理地泛化到完整的厨房。

**消融语言**：为了研究 LLM 的重要性，我们使用语言条件策略进行了两个消融实验。在 BC NL 中，我们将完整指令 i 输入策略——这种方法代表了标准的 RL 或基于 BC 的指令跟随方法。在 BC USE 中，我们通过通用句子编码器（USE）嵌入将高层次指令投影到已知语言命令集中。表 2 中的结果说明了语言落地的必要性，其中 BC NL 在所有任务中实现 0%，BC USE 对单个原语实现 60%，但其他情况为 0%。

**消融价值函数**：表 2 说明了可供性落地的必要性。我们将 PaLM-SayCan 与（1）No VF（移除价值函数落地，即选择最大语言得分技能）和（2）Generative（使用 LLM 的生成输出，然后通过 USE 嵌入将每个计划技能投影到其最大余弦相似度技能）进行比较。No VF 和 Generative 方法表现相似，分别实现 67% 和 74% 的规划成功率，比 PaLM-SayCan 的 84% 更差。

**消融语言模型**：SayCan 能够随着改进的语言模型而改进。本文使用的 LLM 是 PaLM，一个 540B 参数模型。在本节中，我们对 8B、62B 和 540B 参数模型以及 137B 参数 FLAN 模型进行消融。表 3 显示了在机器人上运行完整 SayCan 算法的 PaLM 540B 和 FLAN。结果表明，使用具有可供性落地的 PaLM 的系统（PaLM-SayCan）84% 的时间选择正确的技能序列，74% 的时间成功执行它们，与 FLAN 相比错误减少了一半。这特别令人兴奋，因为这是我们第一次看到语言模型的改进如何转化为机器人技术的类似改进。

#### 5.2 PaLM-SayCan 新能力的案例研究

PaLM-SayCan 实现了新的能力。首先，我们展示了将新技能整合到系统中非常容易，并使用抽屉操作作为示例。其次，我们展示了通过利用思维链推理，我们能够解决需要推理的复杂任务。最后，我们展示了系统可以处理多语言查询，而无需明确设计。

**添加技能：抽屉操作**：SayCan 能够通过简单地将新技能添加为 LLM 的选项并提供相应的价值函数来整合新技能。例如，使用打开、关闭和去抽屉的技能，SayCan 能够解决诸如"将可乐和百事可乐重新放入抽屉"之类的任务。在 21 个查询中，我们发现规划率为 100%，执行率为 33%（由于链式操作策略的失败），其他指令的性能没有损失。

**思维链推理**：SayCan 可以与改进 LLM 推理的最新工作（如思维链）集成。原始 SayCan 的一个局限性是它不处理涉及否定的任务。这是从底层语言模型继承的。然而，我们发现通过使用思维链提示，我们可以在这方面改进 SayCan。对于基于思维链提示的 SayCan，我们需要修改提示以包含称为"解释"的部分。

**多语言查询**：虽然没有明确设计为处理多语言查询，但 PaLM-SayCan 能够处理它们。LLM 在多语言语料库上进行训练，因此 SayCan 可以处理英语以外的多语言查询。SayCan 在多语言查询上的结果总结在表 8 中，当将查询从英语更改为中文、法语和西班牙语时，规划成功率几乎没有下降。

**闭环规划**：如本文所述，SayCan 仅通过当前决策步骤的价值函数接收环境反馈，这意味着如果技能失败或环境发生变化，可能无法获得必要的反馈。由于可扩展性和自然语言接口，Huang 等人在 SayCan 的基础上构建，通过内心独白利用环境反馈（例如来自成功检测器、场景描述符或甚至人类反馈）来实现闭环规划。

#### 6 开源环境

我们在 say-can.github.io/#open-source 的 Google Colab 笔记本中开源了 SayCan 的实现。该环境是一个带有 UR5 机器人和随机生成的彩色积木和碗集的桌面。低层次策略使用 CLIPort 实现，该策略经过训练以输出拾取和放置位置。由于此策略缺乏价值函数，可供性使用 ViLD 对象检测器实现。GPT-3 用作开源语言模型。

#### 7 相关工作

**落地语言模型**：大量工作集中在落地语言上。最近的工作研究了如何落地现代语言模型，通过训练它们接受额外的环境输入或直接输出动作。其他人通过提示工程在环境中落地语言。与 SayCan 同时，Huang 等人使用提示工程提取时间扩展计划，但没有任何额外机制来确保落地，大致对应于我们实验中的"Generative"基线。在我们的工作中，SayCan 能够通过先前训练的价值函数在给定环境中落地语言模型，以零样本方式实现通用的长时域行为，即无需额外训练。

**学习语言条件行为**：研究如何连接语言和行为有着悠久的历史。大量先前工作通过模仿学习或强化学习学习了语言条件行为。这些先前工作中的大多数专注于遵循低层次指令，尽管一些方法在模拟域中处理长时域复合任务。与这些后期工作一样，我们专注于完成时间扩展任务。然而，我们工作的一个核心方面是通过提取和利用大型语言模型中的知识来解决此类任务。

**任务规划和运动规划**：任务和运动规划是对任务进行排序以解决高层次问题的问题，同时确保给定具身体的可行性。传统上，这个问题通过符号规划或优化来解决，但这些需要显式的原语和约束。机器学习最近被应用于实现抽象任务规范、允许通用原语或放松约束。SayCan 利用 LLM 关于世界的语义知识来解释指令并理解如何执行它们。LLM 的使用和学习的低层次策略的通用性使得能够实现长时域、抽象任务，这些任务有效地扩展到真实世界，正如我们的机器人实验所证明的那样。

#### 8 结论、局限性和未来工作

我们提出了 SayCan，一种能够利用和落地大型语言模型中丰富知识以完成具身任务的方法。为了实现真实世界的落地，我们利用预训练技能，然后使用这些技能来约束模型选择既可行又符合上下文的自然语言动作。更具体地说，我们使用强化学习作为学习各个技能的价值函数的方法，这些价值函数提供世界中可能的可供性，然后使用这些技能的文本标签作为由语言模型评分的潜在响应。

这种组合产生了一种共生关系，其中技能及其价值函数可以充当语言模型的"手和眼睛"，而语言模型提供关于如何完成任务的高层次语义知识。我们在多个真实世界机器人任务上评估了所提出的方法，这些任务涉及移动操作机器人在真实厨房中完成大量长时域自然语言指令。我们还展示了我们方法的一个令人兴奋的特性，即机器人的性能可以简单地通过增强底层语言模型来改进。

虽然 SayCan 提供了一种可行的方法来在智能体的可供性中落地语言模型，但它有许多局限性。首先，我们预期该方法会继承 LLM 的局限性和偏见，包括对训练数据的依赖。其次，我们观察到，即使 SayCan 允许用户使用自然语言命令与智能体交互，系统的主要瓶颈在于底层技能的范围和能力。扩展技能库并提高其鲁棒性的未来工作将缓解这一限制。此外，在当前阶段，系统不容易对单个技能尽管报告高价值但失败的情况做出反应，尽管这可能通过适当提示语言模型进行纠正来解决。

还有许多其他潜在的未来工作途径。这项工作提出的一个自然问题是，如何利用通过真实世界机器人经验落地 LLM 获得的信息来改进语言模型本身，无论是在其事实性方面还是在其对真实世界环境和物理进行常识推理的能力方面。此外，由于我们的方法使用通用价值函数来评分可供性，因此考虑可以以相同方式整合哪些其他落地来源是很有趣的，例如非机器人上下文。
