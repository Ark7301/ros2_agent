# Co-NavGPT: 基于视觉语言模型的多机器人协作视觉语义导航

> 来源: [Co-NavGPT: Multi-Robot Cooperative Visual Semantic Navigation Using Vision Language Models](https://arxiv.org/html/2310.07937v3)
> 作者: Bangguo Yu, Qihao Yuan, Kailai Li, Hamidreza Kasaei, Ming Cao (University of Groningen)

---

## 一、研究背景与问题

视觉目标导航（Visual Target Navigation）是自主机器人在未知环境中运行的关键能力。现有方法存在两大不足：

1. 缺乏常识推理能力——传统学习方法依赖大量训练数据，泛化性差
2. 多为单机器人设计——在大型复杂环境中效率低、容错性差

核心问题：如何让多个机器人在未知环境中协作探索，高效定位指定目标物体？

## 二、Co-NavGPT 框架概述

Co-NavGPT 将视觉语言模型（VLM）作为全局规划器，实现无需任务特定训练的多机器人协作导航。

### 系统架构（三层结构）

| 层级 | 功能 | 技术实现 |
|------|------|----------|
| 感知层 | RGB-D观测 → 局部点云 → 全局3D地图 | YOLOv8检测 + Mobile-SAM分割 + DBSCAN去噪 |
| 全局规划层 | VLM分配前沿区域给各机器人 | GPT-4o，多模态提示（视觉+文本） |
| 局部执行层 | 路径规划与动作执行 | Fast Marching Method (FMM) |

### 工作流程

1. 每个机器人采集RGB-D图像，构建局部点云地图
2. 局部地图融合为全局3D语义地图
3. 从2D俯视图中提取前沿区域（frontier）作为候选探索目标
4. VLM根据地图语义信息、机器人状态、目标物体，为每个机器人分配最优前沿
5. 局部策略基于障碍物地图规划路径执行导航

## 三、关键技术点

### 3.1 地图表示

- **3D点云地图**：开放词汇检测器识别物体 → 分割模型提取掩码 → 深度信息投影到全局坐标系
- **2D探索地图**：3D点云投影为俯视2D栅格图（障碍物通道 + 已探索通道），用于前沿提取和路径规划

### 3.2 VLM多模态提示设计

**视觉提示**：全局3D点云渲染为彩色俯视图，标注机器人位置（圆圈+ID）和候选前沿区域（每个前沿单独一张图+ID标签）

**文本提示**包含四部分：
- 任务描述（定位目标物体）
- 上下文背景（多机器人、全局俯视图信息）
- 推理要求（理解场景布局 → 分析协作策略 → 决策分配 → 自我验证）
- 标准化输入输出格式（输出JSON指定各机器人的前沿分配）

### 3.3 决策机制

- VLM仅负责高层目标分配，不做细粒度控制
- 每25个局部步更新一次全局目标
- 分配依据两个准则：①协作探索未知区域 ②语义相关性（前沿是否可能靠近目标）
- 无可用前沿时，随机选择已探索区域内的点作为备选

## 四、实验结果

### 4.1 仿真实验（HM3D数据集）

评估指标：成功率(SR)、路径加权成功率(SPL)、目标距离(DTG)

| 方法 | 特点 | 性能排序 |
|------|------|----------|
| Greedy | 贪心选最近前沿 | 最低 |
| Cost-Utility | 前沿大小与距离的权衡 | 中等 |
| Random Sampling | 随机采样长期目标 | 优于Cost-Utility |
| Multi-SemExp | SemExp扩展到多机器人 | 泛化性差 |
| **Co-NavGPT** | VLM全局规划 | **三项指标均最优** |

### 4.2 消融实验

| 变体 | 结论 |
|------|------|
| 单机器人(Single) | 性能最低，验证多机器人协作的必要性 |
| GPT-4o-mini替代GPT-4o | 性能下降，大模型推理能力对任务有显著贡献 |
| 障碍物图替代语义图(Obs.) | 成功率下降，语义上下文（颜色、布局）对VLM决策很重要 |

### 4.3 真实世界实验

- 平台：2台Unitree Go2四足机器人 + RealSense D455 + Livox Mid-360 LiDAR
- 定位方案：FAST-LIO2（LiDAR-IMU里程计）+ Generalized-ICP（双机器人初始对齐）
- 实时性能：约5 FPS，满足室内多机器人导航需求
- 框架与硬件无关，仅需RGB-D观测和位姿估计作为输入

## 五、失败案例分析

- **探索失败（~14%）**：机器人陷入不完整重建区域、目标不存在或在不同楼层
- **检测失败（~20%）**：误分类或漏检导致提前停止在错误位置

## 六、与本课题的关联价值

1. **VLM作为全局规划器的范式**：证明VLM可以零样本方式协调多机器人任务，无需任务特定训练
2. **多模态提示工程**：视觉提示（标注俯视图）+ 结构化文本提示的设计方法可借鉴
3. **模块化架构思路**：感知-规划-执行三层解耦，VLM仅负责高层决策，局部策略处理细粒度控制
4. **前沿探索机制**：frontier-based exploration与VLM语义推理结合的思路，适用于自主探索场景
5. **真实部署验证**：从仿真到真实四足机器人的迁移路径，包括传感器配置、点云去噪、多机器人坐标对齐等工程细节
