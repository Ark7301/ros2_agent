# 人形机器人"最后一步"实现技术深度调研

> 以"做咖啡"为典型场景，分析人形机器人在非结构化环境中完成复杂操作任务的技术路径

## 1. 问题定义：什么是"最后一步"

人形机器人的"最后一步"指的是：机器人已经具备导航、定位、基本运动能力后，在目标位置完成精细操作任务的能力。以做咖啡为例，这包括：

- 打开抽屉找到杯子
- 抓取咖啡罐并打开盖子
- 精确量取咖啡粉
- 拿起水壶倒水（力控+视觉反馈）
- 搅拌、递送

这些任务的核心难点在于：**长时序（long-horizon）、多步骤、接触丰富（contact-rich）、需要实时适应环境变化**。

## 2. 技术架构总览

"最后一步"的实现通常采用分层架构：

```
┌─────────────────────────────────────────┐
│  高层：任务理解与规划 (LLM/VLM)          │
│  "做一杯咖啡" → 子任务序列               │
├─────────────────────────────────────────┤
│  中层：技能选择与参数化                   │
│  从技能库中选取 grasp / pour / open 等    │
├─────────────────────────────────────────┤
│  低层：运动控制与力反馈                   │
│  关节轨迹生成 + 力/触觉闭环控制           │
├─────────────────────────────────────────┤
│  感知层：视觉 + 触觉 + 力传感器           │
│  物体检测、位姿估计、接触力感知            │
└─────────────────────────────────────────┘
```

## 3. 关键技术路线分析

### 3.1 高层任务规划：LLM/VLM驱动的任务分解

**代表工作：ELLMER框架（爱丁堡大学，Nature Machine Intelligence 2025）**

这是目前最具代表性的"做咖啡"研究。核心思路：
- 使用GPT-4解析抽象指令（如"给我来杯热饮"），结合环境感知生成子任务序列
- 构建知识库（Knowledge Base），存储力-视觉反馈（IFVF）相关的代码模板
- 通过检索增强生成（RAG）为每个子步骤生成定制控制代码
- 每个子步骤都集成力反馈和视觉反馈，实现对扰动的实时适应

论文发表于Nature Machine Intelligence，DOI: 10.1038/s42256-025-01005-x。使用Kinova Gen3机械臂演示了完整的咖啡制作和盘子装饰任务。

**关键创新点：** 将LLM的语义理解能力与底层力-视觉反馈控制结合，而非简单的"LLM出计划 + 机器人盲执行"。

**其他相关工作：**
- RoboDexVLM：VLM驱动的灵巧手任务规划，多模态提示（指令+技能列表+RGB-D图像+记忆）输入VLM生成任务计划
- 子目标树规划（Cross-hierarchical Subgoal Tree）：解决LLM直接生成长序列动作成功率低的问题

### 3.2 中层技能学习：模仿学习与扩散策略

#### 3.2.1 ALOHA系列（Stanford/Google DeepMind）

ALOHA是当前双臂精细操作的标杆系统：
- ALOHA原版：$20k低成本双臂遥操作平台，ACT（Action Chunking with Transformers）策略，10分钟示教数据即可学会穿拉链、装电池等任务，成功率80-90%
- Mobile ALOHA：加入移动底盘，实现全身遥操作
- ALOHA Unleashed：大规模数据采集 + Diffusion Policy，完成挂衣服、系鞋带等高难度双臂任务

**核心思想：** 大量人类示教 + 表达力强的策略模型（Transformer/Diffusion）= 灵巧操作

#### 3.2.2 Diffusion Policy（扩散策略）

将机器人动作生成建模为去噪扩散过程，相比传统行为克隆：
- 能建模多模态动作分布（同一场景可能有多种合理操作方式）
- 对长时序任务更稳定
- 已在ALOHA Unleashed等系统中验证有效

#### 3.2.3 Artly咖啡机器人（商业落地案例，CES 2025/2026）

Artly是目前最接近商业化的机器人咖啡师：
- 使用动作捕捉（MANUS数据手套）记录冠军咖啡师的操作轨迹
- 模仿学习复现完整流程：研磨→压粉→萃取→蒸奶→倒注→拉花
- Intel RealSense深度相机做3D物体感知和位姿估计
- 失败检测+重试机制（抓取失败时回退重来）
- 北美已部署约40台，每次真实执行都作为训练数据回流
- 约2分钟示教即可训练一个窄域技能

**关键启示：** 商业场景中，"窄域精调"比"通用泛化"更实际。

### 3.3 底层基础模型：VLA（Vision-Language-Action）

#### 3.3.1 π0 / π0.5（Physical Intelligence）

Physical Intelligence公司（融资$1.1B，估值$5.6B）的旗舰模型：
- 基于PaliGemma（3B参数VLM）+ Flow Matching动作生成
- 跨机器人形态训练（单臂、双臂、移动机器人）
- 可完成叠衣服、清理桌面、舀咖啡豆等任务
- π0.5进一步引入开放世界泛化能力，首次实现端到端学习系统在全新家庭环境中完成长时序灵巧操作（如清理厨房/卧室）
- 50Hz实时控制

#### 3.3.2 GR-RL / GR-Dexter

- GR-RL：将通用VLA策略通过强化学习精调为长时序灵巧操作专家，多阶段训练流水线（离线行为克隆→动作增强→在线RL）
- GR-Dexter：硬件-模型-数据一体化框架，双臂灵巧手操作，遥操作数据+多模态数据集，在日常操作和泛化抓取上表现强劲

#### 3.3.3 DexVLA

十亿参数级扩散动作专家，专为跨形态长时序任务设计，增强VLA在复杂任务上的效率和泛化能力。

### 3.4 灵巧手与触觉感知（硬件瓶颈）

这是当前公认的最大瓶颈。Rodney Brooks（iRobot创始人）2025年直言："没有任何类人机器人手展示过真正意义上的通用灵巧性。"

#### 3.4.1 F-TAC Hand（Nature Machine Intelligence 2025）

- 高分辨率触觉传感覆盖手部70%表面积
- 实现类人自适应抓取
- 代表了触觉感知的前沿水平

#### 3.4.2 MOTIF Hand（USC，2025）

- 集成热觉、惯性、力传感器的多模态灵巧手
- 提升物体操作中的感知维度

#### 3.4.3 uSkin触觉传感器 + Tesollo DG-5F

- XELA Robotics的3D触觉传感器集成到五指拟人手
- 实现人类级别的抓取和操作精度

#### 3.4.4 灵巧手演进趋势

从浙大2025年综述（arXiv:2507.11840）总结的演进路径：
```
平行夹爪 → 多指欠驱动手 → 刚性灵巧手 → 全驱动拟人手 → 软体手
```
当前主流方案仍是16-22自由度的多指手，配合触觉传感实现力控。

### 3.5 Sim-to-Real强化学习

**代表工作：arXiv:2502.20396（2025）**

在仿真中训练人形机器人灵巧操作策略，直接迁移到真实世界：
- 无需人类示教数据
- 完成抓取-到达、箱子提升、双手交接等任务
- 关键在于域随机化（domain randomization）和仿真保真度

### 3.6 层级混合学习（ARCH框架）

针对长时序接触丰富的装配/操作任务：
- 低层：参数化技能原语库（抓取、插入等），混合RL和基于模型的控制器
- 高层：策略网络选择技能并设定参数
- 比纯端到端模仿学习需要更少的示教数据，且满足高精度要求

## 4. "做咖啡"任务的技术分解

以完整的咖啡制作为例，每个子步骤对应的技术挑战：

| 子任务 | 核心挑战 | 关键技术 |
|--------|---------|---------|
| 理解"做咖啡"指令 | 语义理解、常识推理 | LLM/VLM任务规划 |
| 寻找并打开抽屉 | 未知机构的操作 | 视觉感知 + 力探索 + 自适应控制 |
| 抓取杯子 | 形状多样、易碎 | 触觉反馈 + 力控抓取 |
| 打开咖啡罐 | 旋转操作、双手协调 | 双臂协调 + 灵巧手 |
| 量取咖啡粉 | 精确力控、颗粒物料 | 力传感器 + 视觉估量 |
| 拿水壶倒水 | 重物操作、液体控制 | 力矩控制 + 视觉液位检测 |
| 搅拌 | 工具使用、周期运动 | 轨迹规划 + 阻抗控制 |
| 递送给人 | 人机交互、安全 | 人体检测 + 柔顺控制 |

## 5. 当前主流技术路线对比

| 路线 | 代表 | 优势 | 劣势 | 适用场景 |
|------|------|------|------|---------|
| LLM规划+技能库 | ELLMER | 可解释、模块化 | 技能库需手工构建 | 研究验证 |
| 端到端模仿学习 | ALOHA/ACT | 简单直接、效果好 | 需大量示教数据 | 双臂桌面操作 |
| VLA基础模型 | π0/π0.5 | 泛化能力强 | 精细操作精度不足 | 通用操作 |
| VLA+RL精调 | GR-RL | 兼顾泛化和精度 | 训练复杂 | 长时序灵巧任务 |
| 动捕模仿+窄域精调 | Artly | 商业可落地 | 泛化性差 | 固定场景商业部署 |
| Sim-to-Real RL | 仿真迁移 | 无需真实数据 | sim-to-real gap | 结构化操作 |

## 6. 核心挑战与未解问题

### 6.1 灵巧手的"手问题"
- 人手有27个自由度+数千个触觉感受器，当前最好的机器人手仍远不及
- 触觉传感器的密度、灵敏度、耐久性仍是瓶颈
- 软体/刚性的权衡：太软缺乏精度，太硬缺乏适应性

### 6.2 长时序任务的鲁棒性
- 做咖啡涉及10+个子步骤，每步95%成功率 → 整体仅60%
- 错误累积和恢复策略仍不成熟
- 需要"失败检测+重规划"能力

### 6.3 泛化 vs 精度的矛盾
- 通用VLA模型泛化好但精度不够（倒水量不准、拉花做不了）
- 窄域专家精度高但换个杯子就可能失败
- GR-RL的"通用→专家"精调路线是当前最有前景的折中方案

### 6.4 实时性要求
- 倒水、接触操作需要50Hz+的控制频率
- 大模型推理延迟与实时控制的矛盾
- Physical Intelligence的FAST tokenization方案是一个解决方向

### 6.5 安全与人机共存
- 厨房场景中人会随时出现、移动物品
- 需要柔顺控制 + 碰撞检测 + 力限制
- 当前研究对安全性的关注远不够

## 7. 技术趋势判断

1. **VLA基础模型 + 任务级RL精调** 将成为主流路线（π0.5 + GR-RL的思路）
2. **触觉感知**将从"可选"变为"必选"，F-TAC Hand级别的高密度触觉将成为标配
3. **模仿学习的数据飞轮**：Artly模式（部署→收集数据→改进模型→再部署）将被广泛采用
4. **分层架构不会消失**：即使端到端模型越来越强，高层规划+低层控制的分层仍是工程上最可靠的方案
5. **2-3年内**，固定场景的咖啡制作（如Artly）将大规模商业化；**5年+** 才可能看到通用家庭环境中的可靠操作
